# Monthly Internal Security Review Process

## Purpose

This document defines the process for conducting monthly internal security review meetings.

## Meeting Schedule

Security review meetings should occur once every month.

## Agenda

Typical agenda items include:
- Review previous action items
- Review security incidents
- Review vulnerabilities and patches
- Access control review
- Infrastructure or configuration changes
- Compliance updates
- New risks identified

## Attendees

Meetings should include:
- Engineering representatives
- Security or operations owners
- Relevant stakeholders

## Meeting Minutes

Meeting notes should be stored under:
`docs/security-reviews/minutes/`

## Action Item Tracking

Each action item should include:
- owner
- due date
- status

Follow-up tasks should be tracked using GitHub Issues or project boards.
