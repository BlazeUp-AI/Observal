# Postmortem Process

## When is a Postmortem Required?

A postmortem should be created for:
- Production outages
- Security incidents
- Data corruption or loss
- Customer-impacting failures lasting more than 15 minutes

## Timeline

- Initial incident summary: within 24 hours
- Full postmortem: within 5 business days
- Action items assigned: within 7 days

## Tracking Action Items

- Each action item should have:
  - owner
  - due date
  - status

- Track follow-ups using GitHub Issues or project boards.
